MIPI D-PHY

1 MIPI D-PHY介绍

  • D-PHY的字面意思是设计目标为500Mbits/s的物理接口

  • 两个通过MIPI通信的硬件结构如下图：

    • PHY LAYER为物理层 （如D-PHY）
    • low level protocol和lane management为中间层（如DSI和CSI）
    • application为应用层
    • 通信的最底层由多个lane组成
    • D-PHY支持最少1 x clock lane + 1x data lane,最多1 x clock lane + 4 x data lane
    • clock lane总是单向的，data lane可以双向，但在实际应用中，只有DSI lane0实现为低速双向

  • lane内部结构如下图：

    • LP–low power，主要用于控制(CTRL)或者其他可选的用途
    • HS–high speed，主要用于高速数据(DATA)传输
    • LP和HS均可配置为双向
    • DSI实际配置中，DATA0的LP双向，HS单向，其他的DATA LANE的LP和HS均为单向

  • HS模式和LP模式区别

    HS模式 LP模式
    传输模式 差分信号 单端信号
    传输速度 80M-1.5Gbps <10Mbps
    功耗 大 小

2 主从结构

  • MIPI连接包含一个master设备和一个slave设备，主设备向clock lane提供高速时钟信号，是主要的时钟源；slave设备是主要的设备接受端。从主设备到从设备为正向传输，所有的情况下时钟信号都保持正向传输，只有双向data lane才能反向传输。
  • PPI(phy protocol interfaces)和APPI(abstracted phy protocol interfaces)
    • PHY addapter layer将所有的通道、时钟单元和PPI连接在一起

3 Lane选项

一个PHY包括一个clock lane和一个或者多个data lane，正向传输时所有的data lane都应该支持高速传输和Escape Mode

  • data lane的两种类型

    • 双向通信(bi-directional)(必须包含以下one or both 反向通信选项)

      • 高速反向数据通信
      • low power 反向 escape mode(可选包含或者排除 low power data trassmission(LPDT))

    • 单向通信(undirectional)

    • 所有的lanes都应该包含支持ULPS(ultra low power state休眠模式)和triggers in the forward direction的escape mode

    • 对于双向lane来说，应该为其每个方向设置单独的escape mode

    • PHY配置的选项为：

      1. 单个或多个lane
      2. 双向或者单向lane（对每个lane设置）
      3. 支持的反向通信类型（对每个lane设置）
      4. escape mode支持的功能
      5. 数据传输使用8bit原始数据或者8b9b编码符号

4 lane module types

lane module需要的功能取决于lane的类型和lane处于连接中的（主或从）哪一侧

  • Lane Module Architecture

    • “control and interface logic” – CIL，其功能取决于lane type 和 lane side

  • Lane types

    • 推荐使用的PHY协议接口包括字节格式（也就是前面说的8bit原始数据而不是8b9b编码）的数据输入和输出，输入/输出时钟信号和控制信号。
    • 控制信号包括：
      • 请求
      • 握手
      • 测试
      • 初始化

  • 单向data lane

    • mater module: at least a HS-TX, a LP-TX, and a CIL-MFXN function
    • slave module: at least a HS-RX, a LP-RX and a CIL-SFXN

  • 双向data lane

    双向data lane包含一些反向通信的格式：高速反向通信、reverse escape mode，取决于lane module中包含的反向通信方法

    • Bi-directional Data Lane without High-Speed Reverse Communication 包含以下：
      • a reverse escape mode
      • master module: HS-TX, LP-TX, LP-RX, LP-CD, and CIL-MFXY
      • slave module: HS-RX, LP-RX, LP-TX, LP-CD and CIL-SFXY
    • Bi-directional Data Lane with High-Speed Reverse Communication 包含以下：
      • a reverse Escape mode
      • master module: HS-TX, HS-RX, LP-TX, LP-RX, LP-CD, and CIL-MRXX
      • slave module: HS-RX, HS-TX, LP-RX, LP-TX, LP-CD and CIL-SRXX

  • clock lane

    • master module: HS-TX, LP-TX, and CIL-MCNN
    • slave module: HS-RX, LP-RX and CIL-SCNN
